FP0: Final Project Proposal

Project Concept

My project is a customizable audio visualizer that converts the audio input into visuals.

Sketches

Project Description

Prototype

The prototype of my app will have the desired customizable UI elements and menu that the user will be able to select and change. The user will be able to change display options (light or dark) and theme of the overall layout. In addition, it will allow audio input from the microphone on the device or a specified device.

Final Version

The final version of my app will have the core functionalities mentioned in the prototype Description with a more fleshed out and detailed user interface for the application.

Development Plan

  1. Research APIs and other technologies that I would need to be able to source music from
  2. Make a flowchart diagram for the logic that will dictate the behavior of the back-end
  3. Finalize the design of the UI and its variations
  4. Code the darn thing
  5. Debug and fix any changes
  6. Work towards stretch goals (see below)

Known Unknowns

Some things that I don't know at the moment is how to include different audio inputs. For example, if the audio is not physically playing in the room, or if it's a file, how would that be different. Additionally, if I'm using an API for music files, how would that differ from just having local mp3 files in the project package.

Stretch Goals

Time permitting, my application will include Spotify integration and allow users to sign in to their account to directly use the application as an audio source.
